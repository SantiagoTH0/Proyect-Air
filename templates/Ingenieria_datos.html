Ingeniería de Datos

Selección

  • Fuente de datos: Portal de datos abiertos del gobierno colombiano (datos.gov.co)
  • Tipo de datos: Registros de calidad del aire
  • Ubicación: Estaciones de monitoreo en Pereira
  • Período de tiempo: Registros históricos disponibles
  • Variables seleccionadas: PM10, PM2.5, temperatura, humedad, y otros parámetros de calidad del aire

Limpieza

La limpieza de los datos se realiza de forma automática mediante scripts en Python desarrollados específicamente para este proyecto. Algunas de las correcciones realizadas incluyen:

  • Transformación de la columna tiempo en variables separadas: año, mes, día y hora.
  • Conversión de las columnas de medición (PM10, Temperatura, etc.) a formato numérico.
  • Corrección de errores de codificación UTF-8, como "GIRÃ“N" que fue corregido a "GIRÓN".
  • Estandarización de nombres de estaciones eliminando símbolos y espacios redundantes.
  • Gestión de valores faltantes mediante técnicas de eliminación o imputación.
  • Validación de datos fuera de rango, como temperaturas imposibles (e.g., 1000 grados).

Evaluación de los Datos

  • Estaciones identificadas: Lagos I F/Blanca, La Ciudadela, Santa Cruz Girón, San Francisco y Lagos del Cacique.
  • Columnas bien definidas y claras: estación y tipo de medición.
  • Transformar la columna de tiempo para extraer año, mes, día y hora.
  • Convertir columnas de medición (PM10, temperatura, etc.) a formato numérico válido.
  • Codificar caracteres en UTF-8 para corregir errores como "GIRÃ“N".
  • Estandarizar nombres de estaciones eliminando símbolos y espacios innecesarios.
  • Verificar y tratar valores faltantes.
  • Detectar datos fuera de rango (temperaturas extremas o valores negativos).
  • Reorganizar el dataset para facilitar el análisis: TIEMPO | ESTACION | PM10 | PM2.5 | ...

Transformación

  • Además de las transformaciones mencionadas, se reorganizó el dataset en un formato más manipulable (por ejemplo: TIEMPO | ESTACION | PM10 | PM2.5 | etc.).
  • También se creó una nueva variable llamada calidad_aire, la cual clasifica la calidad del aire según los contaminantes.
  • Posteriormente, las variables originales de contaminación (PM2.5, PM10, NO3, O2) fueron eliminadas para que el modelo comenzara a predecir.

Documentación

Todo el proceso de limpieza y transformación es registrado automáticamente por el sistema. El modelo permite la descarga de un archivo PDF que documenta los cambios realizados sobre el dataset, cumpliendo así con las recomendaciones de ML-Ops sobre trazabilidad y reproducibilidad.
